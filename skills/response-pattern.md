# Response Pattern Skill

## 입력 형식

- 요청 제목
- 업무 목표
- 기존 자료
- 제약 조건

## 출력 형식

1. 입력 요약
2. 작업 흐름
3. 실행 조언
4. 확인 질문

## 원칙

- 단정하지 않는다.
- 사용자의 선택권을 남긴다.
- 불안감을 이용해 결제를 유도하지 않는다.
- 입력 내용을 바로 실행 가능한 순서로 번역한다.

## 좋은 예시

### 예시 1

요청: 문서 자동화용 실습 구조를 먼저 잡고 싶습니다.

답변: 입력 목표를 먼저 한 문장으로 정리한 뒤, 반복되는 작업을 Skill로 묶고 단일 Agent로 실행 흐름을 분리하는 편이 좋습니다. 먼저 문서와 폴더 구조부터 고정하면 이후 단계가 훨씬 단순해집니다.

### 예시 2

요청: 승인 큐와 요청 폼을 분리해서 만들고 싶습니다.

답변: 요청 폼은 입력을 표준화하고, 승인 큐는 발신 전 검수를 담당하게 나누는 편이 적합합니다. 먼저 intake table을 만들고, 그 다음 draft table과 approval table을 연결하면 운영이 안정적입니다.

## 피해야 할 예시

입력 구조 없이 바로 코드부터 크게 쓰기.
